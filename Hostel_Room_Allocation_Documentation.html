Hostel Room Allocation System

A Java Swing Desktop Application with Oracle Database Integration

Language: Java  |  GUI: Swing  |  Database: Oracle  |  IDE: NetBeans

1 Introduction

The Hostel Room Allocation System is a desktop-based application developed using Java (Swing) for the graphical user interface and Oracle Database for persistent data storage. The system is designed to streamline and automate the process of allocating hostel rooms to students in an educational institution.

1.1 Problem Statement

In many educational institutions, hostel room allocation is done manually, which is time-consuming, error-prone, and difficult to track. There is a need for a computerized system that can efficiently manage user authentication, student registration, room allocation, and record-keeping.

1.2 Objectives

The primary objective of this project is to develop a user-friendly application that:

  • Authenticates users through a secure login system with username and password verification.
  • Registers new users by allowing administrators to create accounts with unique IDs, usernames, and passwords.
  • Allocates hostel rooms to students by capturing their student ID, name, and preferred room type (Single, Double, or Triple).
  • Views all room allocations in a tabular format for easy reference and management.

1.5 Database Design

The application uses two Oracle database tables:

📋 USERS
id PK VARCHAR2
username VARCHAR2
password VARCHAR2
🏠 ROOMS
student_id VARCHAR2
student_name VARCHAR2
room_type VARCHAR2

1.6 Application Flow

  1. The application starts with the Login Screen.
  2. New users can navigate to the Registration Screen to create an account.
  3. Upon successful login, the user is directed to the Dashboard.
  4. From the Dashboard, the user can:
    • Navigate to the Room Allocation Form to assign rooms to students.
    • Navigate to the View Allocations screen to see all current room assignments.
    • Logout to return to the Login Screen.
